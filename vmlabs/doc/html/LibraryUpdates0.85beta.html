Library Updates

VM Labs SDK Internal Release 0.86.2
June 14, 2001

The following library updates with this release.

File

Comments
libmutil.a

Version 1.29

  • rewrote _mpedma functions to use _DMALinear
  • Updated documentation
libmml2d.a

Version 2.23

  • The safe-color functions use fixed-point arithmetic only. It is much faster than the prior version.
  • All functions EXCEPT mmlColorFromYCC() return safe colors.
  • Added mmlCustomSafeColorLimits() function, that sets up custom color limits.
  • The mmlSafeColorLimits() function now takes only a single argument, one of the eSafeColorSel constants defined in mml2d.h.
  • Updated documentation.

NOTE: This library is now dependent on libmutil.a; you must include -lmutil and -lm after -lmml2d when linking.
libImpulse.a, libNuonImpulse.a, libImpulseEffects.a, libImpulseDebug.a, libNuonImpulseDebug.a, libImpulseEffectsDebug.a

Now monitors the size of glyphcache.

We now have new documentation for the Impulse libraries in html format.

librombios.a

Version 1.03.15.00

  • Added _VidSetOutputType function
  • Updated documentation

crt0.o, crt01.o, gcrt0.o, gcrt01.o, crtend.o

libg.a, libg_p.a, libc.a, libc_p.a, libm.a, libm_p.a, libstdc++.a, libgcc.a, libio.a, libiostream.a

newlib version 1.7.0 nuon version 10

  • Updated with GCC 2.95.2
  • Fixes rc1 corruption in HALCommHandler
  • Provides video notch filter on/off functionality:

    • _VidNotchFilter(int onoff)

    If onoff == 0, the notch filter is turned OFF. If it is 1, it is turned ON. If it is -1, the system default setting is used -- currently this is ON, but future BIOS/HAL combinations may provide a way for the user to set the default.

  • Correction in gettimeofday for Unix ports
  • Faster versions of memcpy(), memmove(), and bcopy()
libpe.a  
libbios.a Removed. This library is obsolete, and is replaced by librombios.a.
libg2c.a Removed. We don't support Fortran at this time.
