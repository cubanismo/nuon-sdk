February 23, 2001

mGL 3D Graphics Library

The mGL library has been updated to work with the MPE 3 processor and ROM BIOS.  However, your source code will probably need changes with respect to MPE allocation and related issues.

Other changes include:

  • SDRAM-based textures created using mglNewTexture() now work.

  • SDRAM textures can now be created using mglInitJPEGTexture() and mglInitBMPTexture(). An argument was added to these functions to facilitate this. The old prototypes were:

     GLTexture *mglInitJPEGTexture(JOCTET *jpeg_start, GLuint jpeg_size, mmlPixFormat pix, GLint scale);

    GLTexture *mglInitBMPTexture(void *bp, mmlPixFormat pix, GLint scale);

    The new prototypes are:

    GLTexture *mglInitJPEGTexture(JOCTET *jpeg_start, GLuint jpeg_size, mmlPixFormat pix, GLint scale, GLuint sdramFlag);

    GLTexture *mglInitBMPTexture(void *bp, mmlPixFormat pix, GLint scale, GLuint sdramFlag);      

    Applications passing GL_FALSE for sdramFlag will function as before. This modification was applied to the mGL sample programs.
  • The header nuon/gltypes.h has been withdrawn. Its contents are now a part of nuon/gl.h. This makes mGL a bit more like OpenGL. Files which include nuon/gltypes.h should now include nuon/gl.h instead.

 

Please see the mGL documentation for more information about mGL.
